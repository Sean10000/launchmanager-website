macOS task scheduling, done right

Every Mac already has a built-in scheduler called launchd. LaunchManager gives you a visual interface to it — no XML files, no terminal commands, no paid subscription.

On macOS, launchd is the system service that runs scheduled tasks, keeps background processes alive, and starts programs at login or boot. Tools like LaunchControl, Lingon Pro, and LaunchD Task Scheduler are paid front-ends to the same underlying technology. LaunchManager does the same job — browse, create, edit, load, and monitor launchd jobs — but it is free, open source (MIT), and does not require the app to stay running in the background.

Whether you are migrating from crontab, replacing a $30 scheduler app, or setting up your first automated backup script, LaunchManager lets you define schedules through a guided form instead of hand-editing plist XML.

Schedule types launchd supports

⏱ Interval scheduling

Run a script or program every N seconds — ideal for health checks, log rotation, or periodic syncs. Equivalent to a simple cron */5 * * * * job, managed visually.

📅 Calendar scheduling

Run at specific times: daily at 3:00 AM, weekdays at 9:00 AM, or on the 1st of each month. launchd's StartCalendarInterval covers most cron use cases natively on macOS.

🚀 Run at load / login

Start a job when you log in or when the system boots. Perfect for auto-starting dev servers, VPN helpers, or menu-bar utilities without adding Login Items manually.

🔄 Keep alive

Automatically restart a process if it exits or crashes. Useful for local APIs, file watchers, and long-running workers that must stay up 24/7.

📁 Watch paths

Trigger a script when files or folders change. Handy for rebuilding assets, processing uploads, or reacting to config changes without a polling loop.

👤 User, system agent & daemon

Schedule jobs at the right privilege level: per-user agents, system-wide agents, or headless daemons that run without anyone logged in.

Quick start

Four steps to take control of your launchd jobs.

  1. Select a scope from the sidebar: User Agents, System Agents, or System Daemons.
  2. Click + to create a job, or the pencil icon to edit an existing one.
  3. Use row buttons to load, start, or stop a job.
  4. Expand a row (chevron) for details and log output.

Frequently asked questions

Common questions about macOS task scheduling with LaunchManager.

What is launchd and why use it instead of cron?

launchd is Apple's native service manager on macOS. It handles scheduling, process supervision, and resource limits in one system. Apple has deprecated cron in favor of launchd on modern macOS — jobs created with LaunchManager use the supported, first-class API.

Can LaunchManager replace LaunchControl or Lingon?

Yes, for everyday scheduling: creating jobs, setting intervals or calendar triggers, loading/unloading, and viewing logs. If you need an AI assistant or raw XML expert editor, LaunchControl still leads there. For most users who just want reliable scheduled tasks, LaunchManager is a capable free replacement.

Do scheduled jobs keep running after I quit LaunchManager?

Yes. Once a job is loaded into launchd, macOS runs it on schedule independently. LaunchManager is only the configuration tool — it does not need to stay open, unlike some paid schedulers.

Can I schedule shell scripts and Python programs?

Absolutely. Point a job at any executable — shell scripts, Python, Node.js, compiled binaries, or AppleScript wrappers. Set the program path, optional arguments, working directory, and environment variables in the create/edit form.

Is LaunchManager safe and open source?

LaunchManager is MIT licensed. You can read every line of Swift source on GitHub, build it yourself in Xcode, and verify exactly what it does. It uses standard macOS APIs (launchctl) and prompts for admin credentials only when system-level operations require it.

What macOS version do I need?

macOS 14 Sonoma or later. Install via Homebrew (brew install --cask launchmanager), download the DMG from GitHub Releases, or build from source with Xcode 16.
